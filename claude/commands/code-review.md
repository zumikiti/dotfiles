# コードレビューと懸念点評価

指定されたファイルまたはディレクトリを分析し、実装内容の懸念点と緊急性を評価します。

## 評価対象
引数として渡されたファイルパス: `$ARGUMENTS`

## 評価項目と基準

### 1. コード品質 (Code Quality)
- **コード重複**: 同じパターンの繰り返し
- **複雑度**: 関数/クラスの複雑さ
- **可読性**: 命名規則、コメント、構造

### 2. セキュリティ (Security)
- **脆弱性**: 潜在的なセキュリティリスク
- **機密情報**: ハードコーディングされた秘密情報
- **入力検証**: 適切なバリデーション

### 3. パフォーマンス (Performance)
- **処理効率**: アルゴリズムの効率性
- **メモリ使用**: 不必要なメモリ消費
- **I/O処理**: ファイルアクセス、DB接続の最適化

### 4. メンテナンス性 (Maintainability)
- **拡張性**: 新機能追加の容易さ
- **テスト性**: テストの書きやすさ
- **依存関係**: モジュール間の結合度

### 5. 型安全性 (Type Safety)
- **TypeScript準拠**: 型定義の適切性
- **型推論**: 明示的な型定義
- **nullチェック**: 適切なnull/undefined処理

## 緊急性レベル

### 🔴 高緊急度 (Score: 1-3)
- セキュリティ脆弱性
- 本番環境での障害リスク
- 大幅なパフォーマンス劣化
- 開発効率に重大な影響

### 🟡 中緊急度 (Score: 4-7)
- メンテナンス性の問題
- 軽微なパフォーマンス問題
- 型安全性の不備
- コードの可読性問題

### 🟢 低緊急度 (Score: 8-10)
- 軽微な最適化の余地
- コードスタイルの統一
- ドキュメント不足
- リファクタリングの提案

## 出力形式

```
📊 コード品質評価レポート
ファイル: [ファイルパス]
評価日時: [現在時刻]

🔴 高緊急度 (Score: X/10)
- [項目]: [具体的な問題と詳細]
- [改善提案]

🟡 中緊急度 (Score: X/10)
- [項目]: [具体的な問題と詳細]
- [改善提案]

🟢 低緊急度 (Score: X/10)
- [項目]: [具体的な問題と詳細]
- [改善提案]

💡 改善提案:
1. [具体的な改善案1]
2. [具体的な改善案2]
3. [具体的な改善案3]

📈 総合評価: [Score]/10
優先度: [高/中/低]
```

## 指示

1. まず `$ARGUMENTS` で指定されたファイルを読み込む
2. 上記の評価項目に基づいて詳細に分析する
3. 各項目にスコア（1-10）を付与する
4. 緊急性レベルを判定する
5. 具体的で実践的な改善提案を提示する
6. 最終的な総合評価とともに構造化されたレポートを出力する

引数が指定されていない場合は、分析対象のファイルパスを尋ねてください。
